Tutorial 1: Your first Rapture Project 

Objectives:

  1. Use of the RIM user interface to interact with Rapture managed data 
  2. Use of a client-side JAVA app to highlight the Rapture API 

This will cover the following steps:

  1. Uploading a CSV file to a blob repo in your RIM instance
  2. Translating raw CSV data file into document data type
  3. Creating a series using document created in step #2
  4. Running a graph based report from series data in step #3 and saving as blob pdf

For the purposes of this tutorial:

  • A Java sample app is provided to allow you to exercise code and see the output in cmd line and follow along in the RIM user interface
  • It is recommended to also look at the source code
  • In this tutorial mac conventions are used for command line work
  • Windows commands are similar and an App.bat is provided

Before beginning this demonstration of Rapture it is expected that your environment satisfies the following pre-requisites

  1. Java 7 or 8 JRE is installed. At a terminal run java -version to check
  2. If building the Java code on mac
    1. Follow this setup guide for git
    2. brew install gradle

 

 

Create new Repos for use in this Tutorial

 The first step in this tutorial is to login to your RIM instance and create the data repositories.

 

Process

  

Login to your demo environment using url and credentials provided during signup

Create a Document repository using RIM's + Add Repositories interface:
  1. Set Name to tutorialDoc
  2. Set Type to Document
  3. Set Data System to MongoDB
  4. Press the OK Button
 
Create a Blob repository again using RIM's + Add Repositories interface:
  1. Set Name to tutorialBlob
  2. Set Type to Blob
  3. Set Data System to MongoDB
  4. Press the OK Button
 
With the two repositories in place we are ready to move onto the next step in the tutorial: upload a CSV file.

 

To Add a repo simply select "+ ADD" under REPOSITORIES.

 

 

 

Build the project

If you git cloned the project (see main page under Set up your client environment) then please build the Java project.

This will output java executables here:  /Intro01/Java/App/build/install/App/bin (FIX THIS)


$ cd RaptureTutorials/Intro01
$ ./gradlew build installApp

 

 

 

 

Capture Data – Load a CSV

  1. cd RaptureTutorials/Intro01/Java/App/build/install/App/bin (FIX THIS)

  2. Run the Java exe with following parameters

    $ /App -s upload <CSV file location> -p <RIM env password>

At this point we have uploaded the CSV file into the blob://tutorialBlob repository. 

Look at the code in RaptureTutorials/Intro01/Java/App/src/main/java/rapture/tutorial/App.java in the upload() method.

Next up we will run code to translate the CSV file (in the blob repo) to a document (stored in a document repo).

 

(FIX THIS) – no crazy paths like that please.

Translate Data – Create documents from the blob

Again in the same folder Intro01/Java/App/build/install/App/bin run the Java exe with following parameters: (FIX THIS)

 

$ ./App -s blobToDoc -p <RIM env password>

 

At this point the CSV (blob) data has been converted into a document type, which is in this repository: document://tutorialDoc/introDataInbound

Look at the code in RaptureTutorials/Intro01/Java/App/src/main/java/rapture/tutorial/App.java in the blobToDoc() method.

Next up we will look at creating series data from the document.

Analyze Data – Create series data from documents

This step will take the document and create a series 

Again in the same folder Intro01/Java/App/build/install/App/bin run the Java exe with following parameters: (FIX THIS)

 

$ ./App -s docToSeries -p <RIM env password>

 

At this point a a series has been created using data from document://tutorialDoc/introDataInbound.

 Look at the code in RaptureTutorials/Intro01/Java/App/src/main/java/rapture/tutorial/App.java in the docToSeries() method.

Next we will create a pdf from the series repository.

 

$ ./App -s docToSeries -p <your RIM env password>

Distribute Data – create reports from series Data

This step will produce a pdf from the series data.

    $ ./ReportApp
